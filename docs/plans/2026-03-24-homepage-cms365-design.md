# CMS365 风格首页改版设计

## 目标

把 `Interview Atlas` 首页改成更接近 `cms365.cn` 的信息结构、文案节奏和视觉布局，让首页一眼就像“可浏览的面试题库平台”，而不是偏品牌介绍页。

本次改版保留一项现有差异化设计：右上角语言切换按钮继续保留，并维持当前跨语言切换体验。

## 核心方向

- 首页优先解决“易懂、易扫读、易进入内容”
- 借鉴 `cms365.cn` 的首页节奏：Hero 后立刻进入内容分发模块
- 文案风格改成更直接、更平台化、更像题库首页
- 视觉布局贴近竞品的卡片分区和列表感，但不直接复制品牌色

## 页面结构

### 1. 顶部区域

- 保留现有品牌区和右上角语言切换按钮
- 弱化当前偏文档站的导航表达
- 顶部导航更接近平台首页，只保留少量首页内锚点或入口

### 2. Hero

- 使用更直接的大标题，明确站点是 AI / 大模型面试题库站
- 使用一句更结果导向的副标题，强调帮助用户更快进入面试状态
- 视觉上贴近竞品的“大标题 + 短副标题 + 明确平台定位”结构
- 不在 Hero 放太多抽象方法论

### 3. 最新专题

- 模块目标是让用户先看到“这里最近有什么内容”
- 使用首页专用数据展示若干专题卡
- 每张卡展示：专题标题、所属方向、简短说明、入口按钮

### 4. 热门专题

- 模块目标是提供第二层内容入口
- 布局上与“最新专题”平级，形成竞品式双栏目首页节奏
- 内容先允许使用首页专用 dummy 数据

### 5. 热门题目

- 单独做一个信息密度更高的大模块
- 每一行或每张卡展示：所属专题、题目标题、简短分类信息、入口
- 模块阅读体验要更像题库平台，而不是知识站内容块

### 6. 轻量说明区

- 把当前“学习路径”相关内容保留到首页靠后位置
- 用更轻的方式说明为什么这个站的整理方式适合复习
- 该区域不再占据首页第一屏或第二屏

## 数据策略

- 新增 `src/data/homepage.ts` 作为首页专用数据文件
- 提供中英双语首页展示数据
- 包含三组数据：
  - 最新专题
  - 热门专题
  - 热门题目
- 这些数据先允许为 dummy / curated 数据，不要求与现有题目详情页完全自动同步
- 本轮不扩展真实热度字段，不改现有题库数据模型

## 文案策略

- 参考 `cms365.cn` 的平台化表达，但做成 `Interview Atlas` 的变体
- 中文优先使用更直白表达，例如：
  - AI / 大模型面试题库
  - 热门专题
  - 热门题目
  - 帮你更快进入面试状态
- 避免首页继续用“知识地图”“结构优先”等抽象主表达

## 视觉策略

- 模块顺序、分区逻辑、阅读节奏尽量贴近 `cms365.cn`
- 卡片更规整，模块标题更像平台首页栏目
- 保留当前站点的品牌底色，不直接照搬竞品蓝色视觉
- 右上角语言切换继续使用当前设计

## 范围边界

- 本次只改首页相关页面与样式
- 不重构题目详情页和专题详情页
- 不修改现有语言切换逻辑
- 不把首页 dummy 数据接入后端或真实热度系统

## 验收标准

- 用户进入首页后能快速理解这是面试题库站
- 首页首屏后立即能看到清晰的专题和题目入口
- 页面整体阅读节奏明显更接近 `cms365.cn`
- 右上角语言切换保持可用
- `/zh` 与 `/en` 结构一致，只切换文案
